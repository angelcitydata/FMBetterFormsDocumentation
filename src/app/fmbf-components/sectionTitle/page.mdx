# Section Title Component

The Section Title component provides a compact, semantic heading for subsections and field groups in your FM BetterForms application. It features medium typography with optional subtitle support, making it perfect for detailed content organization and establishing clear subsection hierarchy. {{ className: 'lead' }}

    This component is the smallest heading in a family of title components including Page Title and Content Title,
    each designed for specific content hierarchy levels with consistent styling and accessibility.

---

## Component Properties

### Content Properties

<Properties>
  <Property name="title" type="string">
    The section title text to display. This text appears in bold font weight
    with large text size (2xl) and primary foreground color. Used for subsection headings
    and field group titles. Example: `"Personal Details"`, `"Contact Information"`, `"Preferences"`.
    Required property.
  </Property>
  <Property name="subtitle" type="string">
    The optional subtitle text displayed below the main title. This text appears in base text size
    with muted foreground color. Used to provide additional context or description for the section.
    Example: `"Enter your basic information"`, `"Provide your contact details"`.
    Default is empty string.
  </Property>
</Properties>

## Component Structure

The Section Title component provides a compact, semantic heading structure with medium typography and optional subtitle support. It's designed to work as the smallest heading in a content hierarchy alongside Page Title and Content Title components.

### Title Hierarchy

The Section Title component serves as the smallest heading in the content hierarchy:

1. **[Page Title](/fmbf-components/pageTitle)** - Main page heading (largest, most prominent)
2. **[Content Title](/fmbf-components/contentTitle)** - Section and subsection headings (medium size, semibold)
3. **Section Title** - Subsection and field group headings (smaller size) - *Current component*

### Visual Structure

- **Container**: Flex column layout with bottom margin spacing
- **Title Text**: Bold, large text (2xl) with primary foreground color
- **Subtitle Text**: Base text size with muted foreground color (optional)
- **Spacing**: Consistent bottom margin for content separation

### Responsive Design

The Section Title component is designed to be responsive and accessible:

- **Flexible Width**: Titles expand to fill their container width
- **Consistent Spacing**: Standard bottom margin spacing for content separation
- **Accessible Typography**: Proper heading hierarchy and text contrast
- **Semantic HTML**: Uses semantic span elements for proper content structure

## Best Practices

When using the Section Title component, follow these guidelines for optimal user experience and consistent design.

### Content Guidelines

**✅ Do:**

- Keep titles concise and descriptive (1-4 words)
- Use clear, action-oriented titles when appropriate
- Maintain consistent terminology across your application
- Use Section Titles for subsection and field group identification
- Follow the title hierarchy (Page Title → Content Title → Section Title)
- Use subtitles to provide context when helpful

**❌ Don't:**

- Use overly long titles that break the layout
- Mix different terminology for similar content types
- Create titles that don't clearly indicate section purpose
- Use Section Title for main page headings (use Page Title instead)
- Use Section Title for major content sections (use Content Title instead)
- Overuse subtitles when they don't add value

### Content Structure

**Title Guidelines:**
- Use descriptive, clear titles that indicate section purpose
- Keep titles consistent with your application's voice
- Avoid generic titles like "Section" or "Group"
- Use title case for consistency
- Keep titles scannable and easy to understand
- Make titles specific to the content they describe

**Subtitle Guidelines:**
- Use subtitles to provide additional context when helpful
- Keep subtitles concise and descriptive
- Use subtitles to explain the section's purpose or requirements
- Avoid redundant information between title and subtitle
- Use sentence case for readability

### Layout and Design

**✅ Do:**

- Use Section Titles to organize related form fields
- Maintain consistent spacing between title and content
- Align titles with other interface elements
- Consider the visual hierarchy with other title components
- Use Section Titles to establish clear content grouping

**❌ Don't:**

- Create titles that are too wide or too narrow
- Place titles in locations that disrupt content flow
- Use inconsistent spacing between title and content
- Create titles with excessive white space
- Use Section Titles for major content divisions

### Accessibility Considerations

**✅ Do:**

- Ensure sufficient color contrast for title text
- Test with screen readers to ensure proper reading order
- Use descriptive titles that clearly indicate section content
- Maintain consistent heading hierarchy with other title components
- Provide meaningful content structure

**❌ Don't:**

- Rely solely on visual styling to convey information
- Create titles with poor color contrast
- Use generic or unclear titles
- Forget to test with assistive technologies
- Create titles that are difficult to scan

### Performance Tips

**✅ Do:**

- Use Section Titles for static or semi-static content
- Consider the total number of titles on a page
- Use appropriate model keys for title data
- Implement efficient data loading for title content

**❌ Don't:**

- Use Section Titles for frequently changing content without proper state management
- Overload pages with too many titles that impact rendering performance
- Use generic model keys that might conflict with other components
- Create titles that require excessive data fetching
