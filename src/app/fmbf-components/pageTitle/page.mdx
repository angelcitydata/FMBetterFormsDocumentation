# Page Title Component

The Page Title component provides a prominent, semantic heading for main page content in your FM BetterForms application. It features large, bold typography with optional subtitle support, making it perfect for page identification and establishing clear content hierarchy. {{ className: 'lead' }}

    This component is the primary heading in a family of title components including Content Title and Section Title,
    each designed for specific content hierarchy levels with consistent styling and accessibility.

---

## Component Properties

### Content Properties

<Properties>
  <Property name="title" type="string">
    The main page title text to display. This text appears in bold font weight
    with extra large text size (4xl) and primary foreground color. Used for main page headings
    and primary navigation titles. Example: `"User Dashboard"`, `"Settings"`, `"Profile"`.
    Required property.
  </Property>
  <Property name="subtitle" type="string">
    The optional subtitle text displayed below the main title. This text appears in large text size (xl)
    with muted foreground color. Used to provide additional context or description for the page.
    Example: `"Manage your account settings"`, `"View and edit your profile information"`.
    Default is empty string.
  </Property>
</Properties>

---

## Component Structure

The Page Title component provides a prominent, semantic heading structure with large typography and optional subtitle support. It's designed to work as the primary heading in a content hierarchy alongside Content Title and Section Title components.

### Title Hierarchy

The Page Title component serves as the top-level heading in the content hierarchy:

1. **Page Title** - Main page heading (largest, most prominent) - *Current component*
2. **[Content Title](/fmbf-components/contentTitle)** - Section and subsection headings (medium size, semibold)
3. **[Section Title](/fmbf-components/sectionTitle)** - Subsection and field group headings (smaller size)

### Visual Structure

- **Container**: Flex column layout with gap spacing and vertical padding
- **Title Text**: Bold, extra large text (4xl) with primary foreground color
- **Subtitle Text**: Large text (xl) with muted foreground color (optional)
- **Spacing**: Consistent gap and padding for content separation

### Responsive Design

The Page Title component is designed to be responsive and accessible:

- **Flexible Width**: Titles expand to fill their container width
- **Consistent Spacing**: Standard gap and padding spacing for content separation
- **Accessible Typography**: Proper heading hierarchy and text contrast
- **Semantic HTML**: Uses semantic span elements for proper content structure

## Best Practices

When using the Page Title component, follow these guidelines for optimal user experience and consistent design.

### Content Guidelines

**✅ Do:**

- Keep titles concise and descriptive (1-3 words)
- Use clear, action-oriented titles when appropriate
- Maintain consistent terminology across your application
- Use Page Titles for main page identification
- Follow the title hierarchy (Page Title → Content Title → Section Title)
- Use subtitles to provide context when helpful

**❌ Don't:**

- Use overly long titles that break the layout
- Mix different terminology for similar content types
- Create titles that don't clearly indicate page purpose
- Use Page Title for section headings (use Content Title instead)
- Use Page Title for field groups (use Section Title instead)
- Overuse subtitles when they don't add value

### Content Structure

**Title Guidelines:**
- Use descriptive, clear titles that indicate page purpose
- Keep titles consistent with your application's voice
- Avoid generic titles like "Page" or "Home"
- Use title case for consistency
- Keep titles scannable and easy to understand
- Make titles memorable and distinctive

**Subtitle Guidelines:**
- Use subtitles to provide additional context when helpful
- Keep subtitles concise and descriptive
- Use subtitles to explain the page's purpose or scope
- Avoid redundant information between title and subtitle
- Use sentence case for readability

### Layout and Design

**✅ Do:**

- Use Page Titles at the top of each page
- Maintain consistent spacing between title and content
- Align titles with other interface elements
- Consider the visual hierarchy with other title components
- Use Page Titles to establish clear page identity

**❌ Don't:**

- Create titles that are too wide or too narrow
- Place titles in locations that disrupt content flow
- Use inconsistent spacing between title and content
- Create titles with excessive white space
- Use multiple Page Titles on the same page

### Accessibility Considerations

**✅ Do:**

- Ensure sufficient color contrast for title text
- Test with screen readers to ensure proper reading order
- Use descriptive titles that clearly indicate page content
- Maintain consistent heading hierarchy with other title components
- Provide meaningful content structure

**❌ Don't:**

- Rely solely on visual styling to convey information
- Create titles with poor color contrast
- Use generic or unclear titles
- Forget to test with assistive technologies
- Create titles that are difficult to scan

### Performance Tips

**✅ Do:**

- Use Page Titles for static or semi-static content
- Consider the total number of titles on a page
- Use appropriate model keys for title data
- Implement efficient data loading for title content

**❌ Don't:**

- Use Page Titles for frequently changing content without proper state management
- Overload pages with too many titles that impact rendering performance
- Use generic model keys that might conflict with other components
- Create titles that require excessive data fetching
