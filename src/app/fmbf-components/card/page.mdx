# Card Component

The Card component provides a flexible content container for displaying structured information in your FM BetterForms application. It features a clean, accessible layout with title, subtitle, body text, and optional fields, making it perfect for organizing and presenting content in a visually appealing format. {{ className: 'lead' }}

    This component supports structured content display with customizable title and subtitle,
    optional body text, and a flexible field system for organizing related information
    in a consistent, accessible card layout.

---

## Component Properties

### Content Properties

<Properties>
  <Property name="title" type="string">
    The main title displayed at the top of the card. This text appears in bold
    and uses the primary foreground color. Example: `"User Profile"`, `"Order Summary"`.
    Default is empty string.
  </Property>
  <Property name="subtitle" type="string">
    The subtitle displayed below the title. This text appears in smaller font
    size with muted foreground color. Example: `"Personal Information"`, `"Items in cart"`.
    Default is empty string.
  </Property>
  <Property name="bodyText" type="string">
    The main body content of the card. This text appears in regular font size
    with standard foreground color. Only displayed if not empty or whitespace.
    Example: `"Complete your profile to get started"`, `"Review your order details"`.
    Default is empty string.
  </Property>
  <Property name="fields" type="array">
    Array of field objects to display within the card. Each field can contain
    various properties for displaying structured data. Currently supports empty array.
    Example: `[]` (empty array for now, future expansion planned).
    Default is empty array.
  </Property>
</Properties>

## Card Layout Structure

The Card component features a clean, structured layout that organizes content hierarchically. The component automatically adapts its display based on the content provided, showing only relevant sections.

### Content Hierarchy

The card displays content in the following order:

1. **Title** - Main heading displayed at the top in bold text
2. **Subtitle** - Secondary heading below the title in smaller, muted text
3. **Body Text** - Main content area (only shown if content exists)
4. **Fields** - Structured data fields (currently supports empty array)

### Visual Structure

- **Container**: Full-width card with background, border, padding, and rounded corners
- **Header Section**: Contains title and subtitle in a vertical flex layout
- **Body Section**: Contains body text with conditional rendering
- **Fields Section**: Reserved for future field rendering functionality

### Responsive Design

The card component is designed to be responsive and accessible:

- **Full Width**: Cards expand to fill their container width
- **Flexible Height**: Cards adjust height based on content
- **Consistent Spacing**: Standard padding and gap spacing throughout
- **Accessible Typography**: Proper heading hierarchy and text contrast

## Usage Examples

The Card component is configured using JSON objects with properties for content display. Here are common usage patterns for different scenarios.

### Basic Card

<CodeGroup>
```html title="HTML Component"
<bfcomp name="Card" 
        title="Welcome"
        subtitle="Get started with your account"
        bodyText="Complete your profile to begin using all features.">
</bfcomp>
```

```javascript title="Model Definition"
// Define basic card in your model
model.welcomeCard = {
  title: "Welcome",
  subtitle: "Get started with your account",
  bodyText: "Complete your profile to begin using all features.",
  fields: []
}
```

```json title="JSON Configuration"
{
  "title": "Welcome",
  "subtitle": "Get started with your account",
  "bodyText": "Complete your profile to begin using all features.",
  "fields": []
}
```

</CodeGroup>

### How to Add This Component to Your BetterForms App

1. **Navigate to Components**: In your BetterForms site environment, click the **Components** tab in the left sidebar.

2. **Create New Component**: If the Card component is not already in your components list, click the **+New** button at the top left.

3. **Open Component Schema**: In the new component, click the **`</> Component Schema`** button to access the schema editor.

4. **Replace Schema Code**: Copy the complete schema from the **Component Schema** tab above and replace all the existing code in the Component Schema section.


5. **Save Component**: Click the **Save** button in the top right to save the component for use in your application.

6. **Refresh Pages**: **Important**: The component will not be available for use in any live pages you are currently viewing until you manually refresh the page using your browser's Refresh button. However, the component will be immediately available in the **Page Builder** section of every page after you save and navigate away from the Components section.
