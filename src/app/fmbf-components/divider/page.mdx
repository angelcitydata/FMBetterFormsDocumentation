# Divider Component

The Divider component provides visual separation between content sections in your FM BetterForms application. It features multiple border styles and consistent spacing, making it perfect for organizing content and improving visual hierarchy throughout your forms. {{ className: 'lead' }}

    This component supports three variants: solid (default), dashed, and dotted borders,
    each designed for different visual separation needs with consistent spacing and accessibility.

---

## Component Properties

### Visual Properties

<Properties>
  <Property name="variant" type="string">
    The border style variant for the divider. Controls the visual appearance of the divider line.
    Options: `"default"` (solid border), `"dashed"` (dashed border), `"dotted"` (dotted border).
    Default value is `"default"`. Used to create different visual separation styles between content sections.
    Example: `"default"`, `"dashed"`, `"dotted"`.
  </Property>
</Properties>

## Component Structure

The Divider component provides a semantic horizontal rule element with consistent styling and spacing. It's designed to create clear visual separation between content sections with multiple border style options.

### Border Variants

The Divider component supports three border style variants:

1. **Default** - Solid border line (most common, strong separation)
2. **Dashed** - Dashed border line (medium separation, decorative)
3. **Dotted** - Dotted border line (subtle separation, minimal)

### Visual Structure

- **Element**: Semantic `<hr>` element for proper content structure
- **Border**: Configurable border style (solid, dashed, or dotted)
- **Spacing**: Consistent vertical margin for content separation
- **Width**: Full width with self-stretch alignment

### Responsive Design

The Divider component is designed to be responsive and accessible:

- **Full Width**: Dividers expand to fill their container width
- **Consistent Spacing**: Standard margin spacing for content separation
- **Semantic HTML**: Uses semantic `<hr>` element for proper content structure
- **Theme Integration**: Uses theme border colors for consistent styling

### How to Add This Component to Your BetterForms App

1. **Navigate to Components**: In your BetterForms site environment, click the **Components** tab in the left sidebar.

2. **Create New Component**: If the Divider component is not already in your components list, click the **+New** button at the top left.

3. **Open Component Schema**: In the new component, click the **`</> Component Schema`** button to access the schema editor.

4. **Replace Schema Code**: Copy the complete schema from the **Component Schema** tab above and replace all the existing code in the Component Schema section.

5. **Open HTML Tab**: Click the **HTML** tab (located directly next to the Component Schema tab) to access the HTML editor.

6. **Replace HTML Code**: Copy the HTML template from the **HTML Template** tab above and replace all the existing code in the HTML section.

   **Note**: This step might feel redundant since the HTML is included as a key/value in the component schema. However, this is sometimes a quirk in BetterForms, where the HTML doesn't render properly until you manually insert the HTML in the HTML tab on the new component.

7. **Save Component**: Click the **Save** button in the top right to save the component for use in your application.

8. **Refresh Pages**: **Important**: The component will not be available for use in any live pages you are currently viewing until you manually refresh the page using your browser's Refresh button. However, the component will be immediately available in the **Page Builder** section of every page after you save and navigate away from the Components section.

## When to Use Divider

**Use Default Divider for:**
- Strong separation between major content sections
- Form section divisions
- Clear visual breaks in content flow
- Primary content organization

**Use Dashed Divider for:**
- Medium separation between related content groups
- Decorative content breaks
- Subsection organization
- Visual variety in content layout

**Use Dotted Divider for:**
- Subtle separation between closely related content
- Minimal visual breaks
- Content grouping without strong emphasis
- Clean, understated design

## Best Practices

When using the Divider component, follow these guidelines for optimal user experience and consistent design.

### Visual Guidelines

**✅ Do:**

- Use dividers to create clear visual separation between content sections
- Choose appropriate variants based on separation strength needed
- Maintain consistent spacing around dividers
- Use dividers to organize related content groups
- Follow a logical hierarchy (stronger dividers for major sections)

**❌ Don't:**

- Overuse dividers and create visual clutter
- Mix different divider variants inconsistently
- Place dividers too close together
- Use dividers as decorative elements without functional purpose
- Create excessive visual breaks that disrupt content flow

### Variant Selection

**Border Style Guidelines:**
- Use default (solid) for major content section separation
- Use dashed for medium separation between related groups
- Use dotted for subtle separation of closely related content
- Maintain consistency in variant usage across similar content types
- Consider the visual weight and importance of separation needed

### Layout and Design

**✅ Do:**

- Use dividers to organize content into logical sections
- Maintain consistent spacing between dividers and content
- Align dividers with other interface elements
- Consider the visual hierarchy with other layout components
- Use dividers to break up long content sections

**❌ Don't:**

- Create dividers that are too prominent or distracting
- Overcrowd pages with too many dividers
- Use inconsistent spacing around dividers
- Place dividers in locations that disrupt content flow
- Create dividers with excessive visual weight

### Accessibility Considerations

**✅ Do:**

- Use semantic `<hr>` elements for proper content structure
- Ensure dividers provide meaningful visual separation
- Test with screen readers to ensure proper content organization
- Use consistent divider styling for predictable user experience
- Provide clear content structure and organization

**❌ Don't:**

- Rely solely on visual styling to convey information
- Create dividers that are too subtle to be noticed
- Use dividers inconsistently across the application
- Forget to test with assistive technologies
- Create dividers that don't improve content organization

### Performance Tips

**✅ Do:**

- Use dividers for static content organization
- Consider the total number of dividers on a page
- Use appropriate model keys for divider configuration
- Implement efficient rendering for divider components

**❌ Don't:**

- Use dividers for frequently changing content without proper state management
- Overload pages with too many dividers that impact rendering performance
- Use generic model keys that might conflict with other components
- Create dividers that require excessive data fetching
