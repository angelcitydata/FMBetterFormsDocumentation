# Info Card Component

The Info Card component provides a data visualization container for displaying key metrics and statistics in your FM BetterForms application. It features a clean, accessible layout with formatted values, trend indicators, and contextual information, making it perfect for presenting important data points and performance metrics in a visually appealing format. {{ className: 'lead' }}

    This component supports data visualization with customizable titles, formatted values,
    trend indicators with percentage changes, and contextual body text for providing
    additional insights about the displayed metrics.

---

## Component Properties

### Data Properties

<Properties>
  <Property name="title" type="string">
    The main title displayed at the top of the info card. This text appears in smaller
    font size with muted foreground color. Example: `"Average Order"`, `"Total Revenue"`.
    Default is empty string.
  </Property>
  <Property name="value" type="number">
    The primary numeric value to display prominently in the card. This value is
    formatted according to the valueFormat property. Example: `1732`, `45.67`.
    Default is 0.
  </Property>
  <Property name="valueFormat" type="string">
    The format type for displaying the value. Options: `"currency"`, `"percentage"`, or `"number"`.
    Currency uses USD formatting, percentage expects decimal values 0-1, number uses standard formatting.
    Example: `"currency"`, `"percentage"`, `"number"`.
    Default is "number".
  </Property>
  <Property name="percentage" type="string">
    The percentage change value to display in the trend badge. Can be a decimal string
    (e.g., "0.125" for 12.5%) or percentage string (e.g., "12.5%"). Example: `"12.5%"`, `"0.125"`.
    Default is null.
  </Property>
  <Property name="direction" type="string">
    The direction of the trend indicator. Options: `"up"` or `"down"`. Determines
    the arrow icon and color of the percentage badge. Example: `"up"`, `"down"`.
    Default is empty string.
  </Property>
  <Property name="bodyText" type="string">
    The main descriptive text displayed below the value. This text appears in
    semibold font with standard foreground color. Example: `"Trending up this period"`.
    Default is empty string.
  </Property>
  <Property name="bodySubText" type="string">
    Additional contextual information displayed below the body text. This text
    appears in medium font with muted foreground color. Example: `"Compared to $1,539.70 in 2024."`.
    Default is empty string.
  </Property>
</Properties>

## Info Card Layout Structure

The Info Card component features a clean, structured layout that organizes data hierarchically. The component automatically adapts its display based on the content provided, showing only relevant sections.

### Content Hierarchy

The info card displays content in the following order:

1. **Header Section** - Title and trend badge displayed horizontally at the top
2. **Value Section** - Large, prominently displayed formatted value
3. **Body Section** - Descriptive text and contextual information (conditionally rendered)

### Visual Structure

- **Container**: Full-width card with background, border, padding, and rounded corners
- **Header Section**: Contains title and trend badge in a horizontal flex layout
- **Value Section**: Large formatted value display with appropriate number formatting
- **Body Section**: Contains body text and sub-text with conditional rendering

### Responsive Design

The info card component is designed to be responsive and accessible:

- **Full Width**: Cards expand to fill their container width
- **Flexible Height**: Cards adjust height based on content
- **Consistent Spacing**: Standard padding and gap spacing throughout
- **Accessible Typography**: Proper text hierarchy and contrast for data visualization

### Conditional Rendering

Content sections are conditionally rendered based on available data:

```html
<!-- Trend badge condition -->
<span
  v-if="schema.percentage !== null && schema.percentage !== undefined && !isNaN(Number(schema.percentage))"
  class="inline-flex items-center gap-1.5 w-fit select-none text-sm font-medium px-2.5 py-1 border transition-colors rounded-md bg-transparent border-border"
  :class="schema.direction === 'up' ? 'text-success' : 'text-destructive'"
>
  <i
    class="fa-regular"
    :class="schema.direction === 'up' ? 'fa-arrow-trend-up' : 'fa-arrow-trend-down'"
  ></i>
  {{ schema.direction === 'up' ? '+' : '-' }}
  {{ new Intl.NumberFormat('en-US', { style: 'percent', minimumFractionDigits: 1 }).format(Number(schema.percentage) || 0) }}
</span>

<!-- Body text condition -->
<div v-if="!_.isEmpty(_.trim(schema.bodyText))" class="flex flex-col my-4 gap-0.5">
  <span class="text-sm font-semibold text-foreground">{{ schema.bodyText }}</span>
  <span 
v-if="!_.isEmpty(_.trim(schema.bodySubText))" 
class="text-sm font-medium text-muted-foreground whitespace-pre-line"
>
{{ schema.bodySubText }}
</span>
</div>
```

## Best Practices

When using the Info Card component, follow these guidelines for optimal user experience and consistent design.

### Content Guidelines

**✅ Do:**

- Keep titles concise and descriptive (1-4 words)
- Use clear, meaningful titles that indicate the metric type
- Write body text that provides context or insights about the data
- Use body sub-text to add comparison or additional context
- Maintain consistent terminology across your application
- Use info cards to display key performance indicators and metrics

**❌ Don't:**

- Use overly long titles that break the card layout
- Include redundant information between title and body text
- Write body text that doesn't provide additional insights
- Mix different terminology for similar metric types
- Create info cards with unrelated or scattered data

### Content Structure

**Title Guidelines:**
- Use descriptive, metric-focused titles (e.g., "Total Revenue", "Active Users")
- Keep titles consistent with your application's voice
- Avoid generic titles like "Data" or "Metrics"
- Use title case for consistency

**Value Guidelines:**
- Ensure values are properly formatted according to their type
- Use appropriate precision for the data being displayed
- Consider the scale of numbers when choosing format
- Validate that values are meaningful and accurate

**Body Text Guidelines:**
- Write clear, concise insights about the displayed metric
- Use body text to explain trends or provide context
- Keep descriptions short for better readability
- Only include body text when it adds meaningful insights

**Trend Indicator Guidelines:**
- Use trend indicators to show performance changes
- Ensure percentage values are accurate and meaningful
- Choose appropriate direction (up/down) based on context
- Consider the time period for trend calculations

### Layout and Design

**✅ Do:**

- Use info cards to display key metrics and KPIs
- Maintain consistent spacing between cards
- Align cards with other dashboard elements
- Consider the visual hierarchy of multiple cards
- Use info cards to create dashboard layouts
- Group related metrics together
- Use responsive grid layouts (e.g., `grid-cols-1 md:grid-cols-2 lg:grid-cols-4`)
- Implement proper breakpoints for different screen sizes

**❌ Don't:**

- Create cards that are too wide or too narrow
- Overcrowd dashboards with too many cards
- Use inconsistent spacing between cards
- Place cards in locations that disrupt content flow
- Create cards with excessive white space
- Mix info cards with other card types inconsistently
- Forget to test responsive behavior on different devices

### Accessibility Considerations

**✅ Do:**

- Ensure sufficient color contrast for all text elements
- Test with screen readers to ensure proper reading order
- Use descriptive titles that clearly indicate the metric type
- Maintain consistent text hierarchy
- Provide meaningful data structure
- Include ARIA labels for trend indicators

**❌ Don't:**

- Rely solely on visual styling to convey information
- Create cards with poor color contrast
- Use generic or unclear titles
- Forget to test with assistive technologies
- Create cards that are difficult to scan
- Use color alone to indicate trends

### Performance Tips

**✅ Do:**

- Use info cards for metrics that update at reasonable intervals
- Consider the total number of cards on a dashboard
- Use appropriate model keys for metric data
- Implement efficient data loading for metric calculations
- Cache calculated values when appropriate

**❌ Don't:**

- Use info cards for real-time data without proper state management
- Overload dashboards with too many cards that impact rendering performance
- Use generic model keys that might conflict with other components
- Create cards that require excessive data fetching or calculations
- Update cards too frequently without debouncing
