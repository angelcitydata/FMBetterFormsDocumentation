# Content Title Component

The Content Title component provides a simple, semantic heading for content sections in your FM BetterForms application. It features consistent typography and spacing, making it perfect for creating clear content hierarchy and improving readability throughout your forms. {{ className: 'lead' }}

    This component is part of a family of title components including Page Title and Section Title,
    each designed for specific content hierarchy levels with consistent styling and accessibility.

---

## Component Properties

### Content Properties

<Properties>
  <Property name="title" type="string">
    The content title text to display. This text appears in semibold font weight
    with large text size and primary foreground color. Used for content section headings
    and subsection titles. Example: `"User Information"`, `"Contact Details"`, `"Preferences"`.
    Required property.
  </Property>
</Properties>

---

## Component Structure

The Content Title component provides a simple, semantic heading structure with consistent styling and spacing. It's designed to work as part of a content hierarchy alongside Page Title and Section Title components.

### Title Hierarchy

The Content Title component serves as a mid-level heading in the content hierarchy:

1. **Page Title** - Main page heading (largest, most prominent)
2. **Content Title** - Section and subsection headings (medium size, semibold)
3. **Section Title** - Subsection and field group headings (smaller size)

### Visual Structure

- **Container**: Flex column layout with consistent vertical spacing
- **Title Text**: Semibold, large text with primary foreground color
- **Spacing**: Standard margin for content separation

### Responsive Design

The Content Title component is designed to be responsive and accessible:

- **Flexible Width**: Titles expand to fill their container width
- **Consistent Spacing**: Standard margin spacing for content separation
- **Accessible Typography**: Proper heading hierarchy and text contrast
- **Semantic HTML**: Uses semantic span element for proper content structure

### How to Add This Component to Your BetterForms App

1. **Navigate to Components**: In your BetterForms site environment, click the **Components** tab in the left sidebar.

2. **Create New Component**: If the Content Title component is not already in your components list, click the **+New** button at the top left.

3. **Open Component Schema**: In the new component, click the **`</> Component Schema`** button to access the schema editor.

4. **Replace Schema Code**: Copy the complete schema from the **Component Schema** tab above and replace all the existing code in the Component Schema section.

5. **Open HTML Tab**: Click the **HTML** tab (located directly next to the Component Schema tab) to access the HTML editor.

6. **Replace HTML Code**: Copy the HTML template from the **HTML Template** tab above and replace all the existing code in the HTML section.

   **Note**: This step might feel redundant since the HTML is included as a key/value in the component schema. However, this is sometimes a quirk in BetterForms, where the HTML doesn't render properly until you manually insert the HTML in the HTML tab on the new component.

7. **Save Component**: Click the **Save** button in the top right to save the component for use in your application.

8. **Refresh Pages**: **Important**: The component will not be available for use in any live pages you are currently viewing until you manually refresh the page using your browser's Refresh button. However, the component will be immediately available in the **Page Builder** section of every page after you save and navigate away from the Components section.

## Best Practices

When using the Content Title component, follow these guidelines for optimal user experience and consistent design.

### Content Guidelines

**✅ Do:**

- Keep titles concise and descriptive (1-4 words)
- Use clear, action-oriented titles when appropriate
- Maintain consistent terminology across your application
- Use Content Titles to organize related content sections
- Follow the title hierarchy (Page Title → Content Title → Section Title)

**❌ Don't:**

- Use overly long titles that break the layout
- Mix different terminology for similar content types
- Create titles that don't clearly indicate content purpose
- Use Content Title for main page headings (use Page Title instead)
- Use Content Title for field groups (use Section Title instead)

### Content Structure

**Title Guidelines:**
- Use descriptive, clear titles that indicate content purpose
- Keep titles consistent with your application's voice
- Avoid generic titles like "Information" or "Details"
- Use title case for consistency
- Keep titles scannable and easy to understand

### Layout and Design

**✅ Do:**

- Use Content Titles to organize content sections
- Maintain consistent spacing between titles and content
- Align titles with other interface elements
- Consider the visual hierarchy with other title components
- Use Content Titles to break up long content sections

**❌ Don't:**

- Create titles that are too wide or too narrow
- Overcrowd pages with too many Content Titles
- Use inconsistent spacing between titles
- Place titles in locations that disrupt content flow
- Create titles with excessive white space

### Accessibility Considerations

**✅ Do:**

- Ensure sufficient color contrast for title text
- Test with screen readers to ensure proper reading order
- Use descriptive titles that clearly indicate content
- Maintain consistent heading hierarchy with other title components
- Provide meaningful content structure

**❌ Don't:**

- Rely solely on visual styling to convey information
- Create titles with poor color contrast
- Use generic or unclear titles
- Forget to test with assistive technologies
- Create titles that are difficult to scan

### Performance Tips

**✅ Do:**

- Use Content Titles for static or semi-static content
- Consider the total number of titles on a page
- Use appropriate model keys for title data
- Implement efficient data loading for title content

**❌ Don't:**

- Use Content Titles for frequently changing content without proper state management
- Overload pages with too many titles that impact rendering performance
- Use generic model keys that might conflict with other components
- Create titles that require excessive data fetching
